## What was actually fixed

The bug you reported: **portfolio shows 1 FLOW shielded balance duplicated across mUSDC and MockFT rows** (scaled to each token's decimals).

Root cause: `ShieldedCheckpoint.sol` had `mapping(address => Checkpoint)` — singleton per user, one snapshot total. Wrapping mUSDC after FLOW would overwrite the FLOW snapshot. The portfolio code knew about this ("There is ONE checkpoint per COA address — all tokens share this slot") and read once + duplicated across rows.

Why tests missed it: combo-A/B/C/D/E never decrypted the checkpoint — they only checked on-chain `commitments` per Janus contract (which already were per-token). The decrypt → balance round trip was never exercised in multi-token CI.

Fix: per-token mapping `(user, token) → Checkpoint` on EVM. Cadence got the per-token rewrite but the on-chain upgrade was rejected by validator (new field in resource); JanusFT/MockFT remains singleton-coupled — deferred to v0.8.3 governance window.
